# PolyMind UI Guide

PolyMind 的 UI 目标是让用户感觉自己进入了一个高级、灵活、清晰的 AI 思维讨论空间。界面应该表达「多个智能体正在围绕一个问题协作思考」，而不是模拟会议、聊天群或普通后台表格。

## 整体视觉风格

关键词：

- 现代。
- 精致。
- 清晰。
- 有科技感但不过度科幻。
- 信息密度适中。
- 层次丰富。
- 动效柔和。
- 适合长时间阅读。

界面气质：

- 像 Linear 一样有秩序、有速度感。
- 像 Vercel 一样克制、留白明确、边界清楚。
- 像 Stripe 一样有细腻的层次和专业感。
- 像 Framer 一样有轻盈的动效和现代视觉。
- 像 Notion 一样降低内容创作压力。
- 像 Raycast 一样操作直接、反馈迅速。

不要出现：

- 会议室座位图。
- 视频会议风控件。
- 过度拟物的机器人形象。
- 廉价 SaaS 模板首屏。
- 大面积单一紫蓝渐变。
- 随机漂浮光球和无意义装饰。

### Agent 色彩

每个 Agent 可以分配一种识别色，但不要让页面变成彩虹。建议使用低饱和色点、细边框或小型光标标识。

示例：

- Strategist：蓝色。
- Critic：琥珀色。
- Builder：绿色。
- Researcher：青色。
- Writer：玫红色。
- Generalist：中性灰。

## 字体层级

建议使用系统字体栈，优先保证中英文混排清晰：

```css
font-family: Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

层级建议：

- 页面标题：32px 到 40px，字重 650 到 750。
- 空间标题：24px 到 32px，字重 650。
- 区块标题：16px 到 18px，字重 600。
- 正文：14px 到 16px，行高 1.65。
- 辅助信息：12px 到 13px，行高 1.45。
- 按钮文字：13px 到 14px，字重 500 到 600。

原则：

- 不使用负字距。
- 不使用随视口宽度变化的字体大小。
- 长文本区域要优先保证阅读舒适。
- 讨论消息正文行宽不要过长，建议最大 760px。

## 卡片设计

卡片用于表达独立对象或工具面板，例如空间卡片、Agent 卡片、消息块、总结块。不要把整个页面切成一堆厚重卡片，也不要卡片套卡片。

推荐样式：

- 圆角：6px 到 8px。
- 边框：1px 细边框，低对比度。
- 阴影：非常轻，强调层次而不是悬浮感。
- 内边距：紧凑页面 12px 到 16px，重点卡片 20px 到 24px。
- 背景：白色或轻微 tinted surface。
- Hover：边框和背景轻微变化，不跳动。

消息卡片建议：

- 当前流式输出的消息有细微高亮边框。
- Agent 标识固定在消息顶部或左侧。
- 消息正文使用舒适行高。
- 引用其他 Agent 时使用小型引用条，不做复杂嵌套。

## 按钮设计

按钮类型：

- Primary：启动讨论、生成总结、保存。
- Secondary：添加 Agent、继续讨论、复制总结。
- Ghost：编辑、更多、关闭、切换面板。
- Destructive：删除空间、停止讨论。

设计原则：

- 操作按钮配合 Lucide 图标。
- 图标按钮必须有 tooltip 或可访问标签。
- 主按钮不要同时出现多个，避免抢焦点。
- 禁用状态要说明原因，例如缺少 API Key、没有参与者、讨论运行中。
- 启动讨论按钮应有运行态反馈，例如 spinner、状态点或文字变化。

## 动画原则

动效只服务于状态理解和空间感。

适合使用动画的地方：

- 页面和面板进入。
- Sheet、Dialog、Popover 展开。
- 消息流式出现。
- 当前发言 Agent 高亮。
- 讨论状态从 idle 到 running 的切换。
- 总结生成完成后的轻微显现。

参数建议：

- 时长：120ms 到 280ms。
- 缓动：ease-out 或 spring，但阻尼要足。
- 位移：4px 到 12px。
- 透明度变化：轻微即可。

避免：

- 大幅弹跳。
- 长时间循环动画。
- 干扰阅读的背景动画。
- 因动画导致布局跳动。

## 页面布局原则

### 产品首页

首页不做传统营销落地页。第一屏应是产品入口：

- 左侧或中心是创建讨论空间的主要输入。
- 下方露出场景模板或最近空间。
- 顶部导航简洁。
- 视觉重点是「开始一次 AI 讨论」。

### 空间详情

推荐桌面布局：

```text
┌──────────────────────────────────────────────┐
│ Space header: title, status, actions          │
├──────────────┬───────────────────┬───────────┤
│ Agents panel │ Discussion stream │ Context   │
│              │                   │ Summary   │
└──────────────┴───────────────────┴───────────┘
```

宽度建议：

- 左侧 Agent 面板：240px 到 300px。
- 中心讨论流：弹性，占主要空间。
- 右侧上下文栏：300px 到 360px。

移动端：

- 默认展示讨论流。
- Agent 和上下文进入 Tab、Drawer 或 Sheet。
- 核心操作固定在顶部或底部安全区域。

### 讨论流

讨论流是核心体验，不应只是普通聊天。

必须表达：

- 当前轮次。
- 当前发言者。
- 发言状态。
- Agent 之间的回应关系。
- 讨论是否正在收敛。

可使用：

- 轮次分隔线。
- Agent 色点。
- 当前发言脉冲边框。
- 消息状态标签。
- 轻量引用条。

## 参考网站风格分析

### Apple

可借鉴：

- 高级留白。
- 对齐纪律。
- 内容本身成为视觉中心。

不可照搬：

- 过度产品摄影式页面。
- 大段营销叙事。

### Linear

可借鉴：

- 深色模式质感。
- 紧凑信息布局。
- 快速操作感。

不可照搬：

- Issue 管理范式不能直接套到讨论空间。

### Vercel

可借鉴：

- 克制色彩。
- 清晰边界。
- 工程师友好的冷静界面。

不可照搬：

- 过度黑白极简导致产品缺少情绪。

### Stripe

可借鉴：

- 复杂能力的清晰分层。
- 精致渐变和细腻阴影。

不可照搬：

- 大面积品牌渐变。

### Framer

可借鉴：

- 页面切换和局部交互动效。
- 现代、轻盈、有设计感的界面节奏。

不可照搬：

- 过度展示型页面。

### Notion

可借鉴：

- 低压力输入体验。
- 内容组织方式。
- 简洁面板。

不可照搬：

- 过于平淡的默认文档感。

### Raycast

可借鉴：

- 快速命令感。
- 键盘友好。
- 操作反馈明确。

不可照搬：

- 完全命令面板化，PolyMind 仍需要可阅读的讨论空间。

## 禁止粗糙、模板化、廉价感 UI

明确禁止：

- 默认 create-next-app 首页不改就交付。
- 大面积无语义渐变背景。
- 卡片阴影过重。
- 圆角过大导致像玩具界面。
- 按钮没有状态反馈。
- 图标风格混乱。
- 表单像后台管理系统。
- 空状态只写一句「暂无数据」。
- 用机器人插画替代真实产品体验。
- 所有 Agent 消息长得完全一样。
- 讨论空间里出现摄像头、麦克风、会议成员席位等会议软件隐喻。

质量标准：

- 每个主页面都应该能截图作为产品展示。
- 每个可点击元素都有 hover、active、disabled 或 loading 状态。
- 每个核心流程都有空状态、错误态和成功反馈。
- 讨论流在无数据、运行中、暂停、失败、结束时都有清晰表现。
